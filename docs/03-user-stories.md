# User Stories

## Player

### US-001: Player Registration

As a player,
I want to create an account,
so that I can book sports courts through the platform.


Acceptance Criteria:

- Player can provide registration details.
- Player account is created successfully.
- Player can log in after registration.


---

### US-002: Search Venues

As a player,
I want to search for available venues,
so that I can find courts suitable for playing.


Acceptance Criteria:

- Player can browse available venues.
- Player can view venue information.
- Player can view available courts.


---

### US-003: Book a Court

As a player,
I want to reserve a court,
so that I can schedule a playing session.


Acceptance Criteria:

- Player selects a court.
- Player selects an available time.
- System validates availability.
- Booking is created.


---

## Venue Owner

### US-004: Register Venue

As a venue owner,
I want to register my venue,
so that players can discover and book my courts.


Acceptance Criteria:

- Venue owner can provide venue details.
- Venue is linked to the owner's account.


---

### US-005: Manage Courts

As a venue owner,
I want to add and manage courts,
so that my venue availability is accurate.


Acceptance Criteria:

- Owner can create courts.
- Owner can edit court details.
- Owner can remove courts.


---

## Administrator

### US-006: Manage Platform

As an administrator,
I want to manage users and venues,
so that the platform remains reliable.
